# 任务概述
- 目标：完善 `.gitignore`，避免 UE 与 UnrealCSharp/C# 工具链的生成物被提交到远端仓库，从而减少“源码管理”里的噪音变更。
- 动机：当前出现大量未跟踪/可再生成文件（例如 `Content/Script/`、`Script/**/bin/obj/` 等），影响日常提交与审查。
- 影响范围：仅更新 `.gitignore`（不对现有已跟踪文件做删除/改名）。

# 聊天与讨论背景
- 用户反馈：Git 源码管理里出现大量挂起变更，希望通过 `.gitignore` 避免不必要文件上传远端。
- 关键约束：只处理忽略规则；不擅自对已跟踪文件执行 `git rm --cached` 等破坏性操作。

# 设计思路
- 以“可再生成”为判断标准：忽略 UE 常见生成目录（`Saved/`、`Intermediate/`、`DerivedDataCache/`）以及插件的构建输出。
- 针对 UnrealCSharp：保留插件源码（submodule），只忽略其 `Binaries/Intermediate/Saved` 等输出目录。
- 针对 C# 工具链：忽略 `Script/**/bin/`、`Script/**/obj/`、`Script/**/Proxy/`、以及 `Content/Script/`（C# 产物输出目录）。

# 关键改动说明
- `.gitignore`：
  - 修正 `Plugins/UnrealCSharp/*` 的过度忽略（会掩盖插件源代码/结构），改为仅忽略 `Plugins/UnrealCSharp/**/Binaries|Intermediate|Saved`。
  - 增加 C#/.NET 生成物忽略：`Script/**/bin/`、`Script/**/obj/`、`Script/**/Proxy/`、`Content/Script/` 等。
  - 增加 UnrealCSharp 本地工具链工程忽略（本仓库 HEAD 未版本化）：`Script/SourceGenerator/`、`Script/Weavers/`、`Script/UE/`、`Script/CodeAnalysis/`、`Script/Game/*.csproj*`、`Script/Game/FodyWeavers.*` 等。
  - 补充 `Saved/`、`Intermediate/`、`DerivedDataCache/` 的目录型忽略写法，并扩展插件目录匹配到 `Plugins/**/`。
