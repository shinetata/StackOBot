# 设计思路
- 用“调用形态 + 返回语义 + 常见边界”组织内容：
  - 属性访问（`FPropertyImplementation.*`） vs 函数调用（`FFunctionImplementation.*`）。
  - 返回值分为“值类型拷贝”与“UObject/Actor 代理 + 句柄”。
  - 强调生命周期收尾（解绑委托/取消 async）与线程边界（UE API 通常要求 Game Thread）。
- 结合本仓库已有真实示例做锚点（不引入新的结构/框架）。
