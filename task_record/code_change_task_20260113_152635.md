# 任务记录：为 C# 暴露 UE native thread id（用于验证 TaskGraph worker 分布）

## 任务概述
- **目标**：给 C# 侧提供一个可直接获取“当前 UE native thread id”的接口，用于验证多个 TaskGraph 任务是否分布在不同 worker 线程上执行。
- **动机**：现有 PoC 里 `GT != Worker` 是通过 C++ 侧采集并传参到 C# 打印的；但批次执行（`TaskGraphBatch.ExecuteBatch`）希望 C# 侧可自行记录每条日志来自哪个 worker，需要一个通用的 `GetCurrentThreadId`。

## 关键改动说明
- 新增 internal call：`FTaskGraph.GetCurrentThreadId`
  - C++ 实现直接返回 `FPlatformTLS::GetCurrentThreadId()`。
- C# 侧新增 extern 并封装成易用方法：
  - `FTaskGraphImplementation.FTaskGraph_GetCurrentThreadIdImplementation()`
  - `TaskGraphBatch.GetCurrentNativeThreadId()`

## 验证方式与结果
- 在任意会执行的 C# 入口调用（示例）：
  - `TaskGraphBatch.ExecuteBatch(i => Console.WriteLine($"i={i} nativeTid={TaskGraphBatch.GetCurrentNativeThreadId()} managedTid={Thread.CurrentThread.ManagedThreadId}"), 8);`
- 预期：Output Log 中应出现 `nativeTid`，且可能观察到多个不同的 `nativeTid`（不保证 8 个都不同）。
