# 任务概述
- 目标：新建一份“从项目启动到 C# 执行”的运行时执行流程文档，模拟 UE 运行时如何在 GameThread 拉起 Mono、何时进入 Mono 执行 C#、何时回到 UE，并给出清晰的代码级流程图。
- 动机：为团队建立统一的、可对照源码的“整体执行心智模型”，便于排障与协作（尤其是 C#-First 使用方式下）。
- 影响范围：新增文档与索引/记录，不修改运行时代码。

# 聊天与讨论背景
- 用户需求：
  - 需要“详细地代码流程图”，从项目启动开始模拟整个执行路径；
  - 不使用 Mermaid，图形风格参考 `ScheduleParallel_非阻塞并行化设计说明.md` 的 ASCII 架构图。

# 设计思路
- 以“模块加载 → 激活 → Mono 初始化 → 每帧 Tick → 事件回调 → 异步加载绑定 → 退出回收”为主线。
- 图内直接标注关键文件与关键函数，确保读者能“一眼跳到代码”：
  - `FEngineListener::SetActive`、`FUnrealCSharpCoreModule::SetActive`
  - `FCSharpEnvironment::Initialize`
  - `FDomain::Initialize` / `FMonoDomain::Initialize`
  - `FDomain::Tick` + `SynchronizationContext.Tick`
  - `FCSharpDelegateDescriptor::CallDelegate` / `mono_runtime_invoke_array`
  - `FCSharpEnvironment::OnAsyncLoadingFlushUpdate` / `Object_Constructor`
- 强调“同线程执行”与“跨线程派发”两种调度形态，并引用 C# 侧同步上下文实现文件作为证据。
