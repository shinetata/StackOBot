# code_change_task_20260121_162748

## 1. 任务概述
- 目标：修正 UE::Tasks thunk 直调的调用签名，补齐异常输出参数。
- 动机：避免 thunk 调用栈不匹配导致崩溃，并保留异常处理能力。
- 影响范围：仅 UE::Tasks internal call 逻辑。

## 2. 聊天与讨论背景
- 用户反馈 Play 闪退，并提供 mono cooperative GC 状态错误日志。

## 3. 设计思路
- 参照 `SynchronizationContextTick` 的 thunk 使用方式，补齐 `MonoObject**` 异常输出参数。
- 仍保持不使用 Runtime_Invoke。

## 4. 关键改动说明
- thunk 签名从 `void(void*, int32, int32)` 调整为 `void(void*, int32, int32, MonoObject**)`。
- 调用后检查异常并走 `Unhandled_Exception`。

## 5. 关键代码解析
- `FExecuteTaskThunk` 新增异常输出参数，调用后处理异常指针。
