# 任务概述
- 目标：为 ScheduleUeParallel 增加入口测试（功能+性能），并修复 Combine 相关 GCHandle 生命周期管理。
- 动机：需要验证非阻塞与性能，同时避免合并时提前释放 GCHandle。
- 影响范围：UETasksJobHandle、UETasksJobHandleUtils、MainMenu_C 测试入口。

# 聊天与讨论背景
- 用户要求补入口测试，且包含性能对比。
- 发现 Combine 立即 Dispose 旧 handle 会提前释放 GCHandle，需要调整。

# 设计思路
- UETasksJobHandle 支持持有多个 GCHandle，Combine 时转移所有权。
- UETasksJobHandleUtils.Combine 改为 DetachForCombine 收集 GCHandle，创建 combined handle。
- MainMenu_C 增加 ScheduleUeParallel 与 Combine 的性能测量。

# 关键改动说明
- UETasksJobHandle：新增 extraHandles 管理、DetachForCombine、CreateCombined。
- UETasksJobHandleUtils.Combine：不再 Dispose 旧 handle，转移 GCHandle 所有权。
- MainMenu_C：新增 ScheduleUeParallel 与 Combine 的性能测试入口。

# 关键代码解析
- DetachForCombine：将 GCHandle 转移到 combined handle，避免提前释放。
- CreateCombined：创建持有多个 GCHandle 的新句柄。
- RunParallelPerf：新增 MeasureSchedule/MeasureScheduleCombine。

# 涉及文件清单

  - 支持合并场景的 GCHandle 管理。
- Script/Game/PGD/Parallel/UETasksJobHandleUtils.cs
  - Combine 使用 DetachForCombine 并创建 combined handle。
- Script/Game/Game/StackOBot/UI/MainMenu/MainMenu_C.cs
  - 新增 ScheduleUeParallel/Combine 性能测试入口。
