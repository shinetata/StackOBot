# 任务记录：改为按 chunk 直接调度 UE Tasks

## 1. 任务概述
- 目标：移除 C# 内部动态调度逻辑（Interlocked），按 chunk 直接调度 UE Tasks。
- 范围：简化 ExecuteTask 逻辑与任务数计算。

## 2. 聊天与讨论背景
- 用户要求：不要 C# 额外调度逻辑，按顺序把 chunk 交给 UE Tasks。

## 3. 设计思路
- 任务数固定为 chunk 数，每个任务只处理一个 chunk。
- 去掉 Interlocked 动态抢 chunk。

## 4. 关键改动说明
- `ExecuteUeParallel` 使用 `count` 作为任务数。
- `QueryRunner.ExecuteTask` 仅处理当前索引对应的 chunk。
- 删除 `ResolveTaskCount` 与 `GetDefaultTaskCount`。

## 5. 关键代码解析
- C++ 仍负责调度任务；C# 只按索引处理 chunk。
