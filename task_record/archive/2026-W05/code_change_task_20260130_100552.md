# 任务概述
- 目标：为 UE Tasks 的 ScheduleUeParallel 增加依赖合并与链式调度能力，减少 List 遍历与等待开销。
- 动机：参考 Unity ECS 依赖模型，引入 handle 合并与 then 调度。
- 影响范围：UnrealCSharp 内部调用、PGD 扩展工具类。

# 聊天与讨论背景
- 用户希望提供链式传递 + 合并依赖能力，避免大量 List 处理。
- 约束：静态工具类接口、Native 侧支持 Combine。

# 设计思路
- C++ 侧新增 CombineHandles：合并多个 handle 对应任务列表并返回新 handle。
- C# 侧新增 UETasksJobHandleUtils 提供 Combine/Then。
- UETasksJobHandle 暴露内部 handleId 供合并使用。

# 关键改动说明
- FTasksQuery.cpp：新增 CombineHandles InternalCall。
- FTasksQueryImplementation.cs：新增 CombineHandles 声明。
- UETasksJobHandle.cs：新增内部 HandleId 访问器。
- UETasksJobHandleUtils.cs：新增 Combine 与 Then 静态方法。

# 关键代码解析
- Combine：收集 handleId，native 合并后释放旧 handles。
- Then：Wait + Dispose 依赖后 schedule 新任务。
