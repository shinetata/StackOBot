# 任务概述
- 目标：新增 UE Tasks 的非阻塞调度接口（ScheduleParallel 风格），返回可等待的 JobHandle。
- 动机：参考 PGD 的 ScheduleParallel，允许调用方自行决定等待时机。
- 影响范围：UnrealCSharp 内部调用与 PGD 扩展方法。

# 聊天与讨论背景
- 需求：不阻塞主线程，暴露类似 JobHandle 的等待方式。
- 约束：Debug 模式下同步执行并返回已完成 handle。

# 设计思路
- C++ 侧新增 ScheduleBatch + handle 管理（完成查询、等待、释放）。
- C# 侧新增 UETasksJobHandle，提供 Wait/IsCompleted/Dispose。
- PGD 扩展新增 ScheduleUeParallel，返回 UETasksJobHandle。

# 关键改动说明
- FTasksQuery.cpp：新增 ScheduleBatch/IsHandleCompleted/WaitHandle/ReleaseHandle，并维护任务句柄表。
- FTasksQueryImplementation.cs：新增 InternalCall 绑定。
- UETasksJobHandle.cs：封装 handle 的等待与释放。
- UETasksQueryExtensions.cs：新增 ScheduleUeParallel 重载。

# 关键代码解析
- ScheduleBatch：Launch UE::Tasks 后将任务数组存入 HandleTasks，返回 handleId。
- WaitHandle：拷贝任务数组等待完成。
- ReleaseHandle：移除 handle，并等待任务完成以安全释放。
- UETasksJobHandle：Dispose 时释放 native handle 与 GCHandle。
