# code_change_task_20260121_191408

## 1. 任务概述
- 目标：让 UE::Tasks 方案A支持开发者在 C# 侧自定义 slice 处理逻辑，无需改 C++。
- 动机：用户反馈 ExecuteSlice 写死，缺乏解耦。
- 影响范围：仅 UETasksSliceBatch 的托管逻辑与调用方式。

## 2. 聊天与讨论背景
- 用户强调开发者不会关注 C++ 逻辑，需要在 C# 侧定义处理逻辑。

## 3. 设计思路
- 引入可注入的 Managed/Native slice handler（Action）。
- ExecuteSlice 只负责分发到当前注册的 handler。
- 保留现有 AddOneAndSum 作为默认测试实现。

## 4. 关键改动说明
- 新增 RunManaged/RunNative 入口支持自定义 handler。
- ExecuteSlice 根据模式调用对应 handler。
- 运行前配置并固定 handler，避免 C++ 侧改动。

## 5. 关键代码解析
- ConfigureManaged/ConfigureNative 设置数据与 handler。
- ExecuteSlice 根据 Mode 调用 handler 并完成 slice 处理。

## 6. 涉及文件清单
- `Plugins/UnrealCSharp/Script/UE/Library/UETasksSliceBatch.cs`
  - 新增 handler 注入与执行分发。

## 7. 验证方式与结果
- 未执行（需用户调用 RunManaged/RunNative 并观察日志/结果）。
