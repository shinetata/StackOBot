# 任务记录：拆分 TaskGraph 批次接口为 baseline/testline 两条线（便于对比与回填）

## 任务概述
- **目标**：实现“改动前 vs 改动后”的真实接口对比：baseline 与 testline 走不同 internal call 入口，便于只改 testline 并用同一套 benchmark 对比；确认提升后再回填 baseline。
- **原因**：仅在托管侧写两份 `ExecuteIndex` 无法隔离 C++/桥接层优化（两条线会同时受益，无法对比）。

## 关键改动说明
- C++ 新增两条 internal call 入口（目前行为与旧版一致）：
  - `FTaskGraph.ExecuteBatchBaseline`
  - `FTaskGraph.ExecuteBatchTestline`
  - 三者（Baseline/Testline/原 ExecuteBatch）复用同一个内部调度函数 `ExecuteBatchInternal(..., ManagedClassName)`，差异在于回调到的托管类不同。
- C# 新增两条批次桥接类：
  - `TaskGraphBatchBaseline` / `TaskGraphBatchTestline`
  - 各自持有自己的 `BatchState` 类型，并对应各自的 `ExecuteTask(handle, index)`。
- 性能对比 Harness 改为按“接口线”对比：
  - baseline run 调 `TaskGraphBatchBaseline.ExecuteBatch`
  - testline run 调 `TaskGraphBatchTestline.ExecuteBatch`

## 验证方式与预期结果
- 进入 MainMenu 关卡触发 `MainMenu_C.ReceiveBeginPlay`。
- Output Log 应出现两条总耗时输出：
  - `[TaskGraphPerf] baseline total=...ms`
  - `[TaskGraphPerf] testline total=...ms`
