# 任务记录：修复 TaskGraph 批次等待的类型不匹配编译错误

## 任务概述
- **目标**：修复重新生成 UE 工程时报错 `WaitUntilTasksComplete` 参数类型不匹配的问题，保证 UnrealCSharp 插件可正常编译。
- **现象**：编译错误提示无法将 `TArray<FGraphEventRef>` 转换为 `const FGraphEventArray&`。

## 原因分析
- UE 的 `FTaskGraphInterface::WaitUntilTasksComplete` 形参类型是 `const FGraphEventArray&`（而不是任意 `TArray<FGraphEventRef>`）。
- 我们在 `ExecuteBatchImplementation` 中用 `TArray<FGraphEventRef>` 收集事件，导致类型不匹配。

## 关键改动说明
- 将事件数组类型从 `TArray<FGraphEventRef>` 改为 `FGraphEventArray`，保持与 UE API 签名一致。

## 验证方式与结果
- 在 Visual Studio / UBT 重新生成 `StackOBotEditor Win64 Development`。
- 预期：不再出现 `C2664 WaitUntilTasksComplete` 的类型转换错误。

## 涉及文件清单
- `Plugins/UnrealCSharp/Source/UnrealCSharp/Private/Domain/InternalCall/FTaskGraph.cpp`
  - `Events` 类型改为 `FGraphEventArray`
